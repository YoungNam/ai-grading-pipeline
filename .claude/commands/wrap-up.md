오늘 세션에서 작업하고 변경된 최종 내용을 정리하고 GitHub에 반영한다.
아래 3단계를 순서대로 수행하라.

---

## Step 1 — handoff.md 업데이트

`/Users/justin0/ai_Projects/handoff.md` 를 읽고, 오늘 변경된 내용을 반영하여 업데이트한다.

업데이트 기준:
- 최종 업데이트 날짜를 오늘로 갱신
- 새로운 버그 수정은 "버그 수정 이력" 섹션에 추가
- 새로운 설계 결정/아키텍처 변경은 해당 섹션에 추가 또는 수정
- 미완성 항목이 완료되었으면 제거
- 새로운 미완성 항목이 생겼으면 추가
- 기존에 없는 중요한 구현 패턴이나 설계 원칙이 확인되었으면 추가
- 이미 문서에 있는 내용은 중복 작성하지 않음

## Step 2 — claude.md 폴더 업데이트

`/Users/justin0/ai_Projects/claude.md/` 폴더 내의 `.md` 파일들을 확인하고,
기술 스택, UI 명세, 시스템 아키텍처 중 오늘 변경된 부분이 있으면 업데이트한다.

업데이트 기준:
- 실제로 변경이 필요한 경우에만 수정 (불필요한 수정 금지)
- 구현 명세와 실제 구현이 달라진 부분을 동기화

## Step 3 — MEMORY.md 업데이트

`/Users/justin0/.claude/projects/-Users-justin0-ai-Projects/memory/MEMORY.md` 를 읽고
오늘 확정된 패턴, 설계 원칙, 주요 버그 수정 내용 중 다음 세션에도 기억해야 할 것을 반영한다.

업데이트 기준:
- 이미 있는 내용의 중복 작성 금지
- 틀린 내용이 있으면 수정
- 200줄 이내로 유지 (초과 시 핵심 내용만 남기고 정리)

## Step 4 — GitHub Push

아래 순서로 git 작업을 수행한다:

1. `git status` 로 변경된 파일 확인
2. 변경된 파일 전체를 스테이징 (단, `.env`, 평문 API 키 파일은 절대 포함하지 않음)
3. 아래 형식으로 커밋 메시지 작성:
   - 제목: 오늘 세션의 핵심 변경 내용을 한 줄로 요약 (한국어)
   - 본문: 변경 사항을 카테고리별로 간략히 나열
   - 마지막 줄: `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>`
4. `git push origin master`

모든 단계 완료 후 수행한 작업을 한국어로 간략히 요약하여 보고한다.
