## 1. 프로젝트 개요 (Overview)

본 프로젝트는 규칙 기반(Rule-base) 엔진과 다중 LLM 앙상블(Multi-LLM Ensemble) 아키텍처를 결합하여
한국어 서논술형 문항, 수식, 과학 논술을 자동 채점하고 피드백을 생성하는 지능형 파이프라인을 구축합니다.
교사가 AI 결과를 검수·조정할 수 있는 HITL(Human-in-the-Loop) 웹 대시보드를 함께 제공합니다.

---

## 2. 기술 스택 (Tech Stack)

### 백엔드 파이프라인
- **핵심 워크플로우**: Python, LangGraph (노드 기반 LLM 에이전트 상태 관리)
- **API 서버**: FastAPI + Uvicorn, SSE(Server-Sent Events) 스트리밍

### 과목별 Rule-base 엔진

| 과목 | 엔진 | 평가 방식 |
|------|------|-----------|
| **수학** | SymPy | 자연어 답안에서 수식 후보 추출 → `simplify()` 대수 동치 → `evalf()` 수치 동치 → `solve()` 방정식 해집합 비교. **단일 동치성 결과(0 or 1) → 전체 기준 균등 적용** (키워드 매칭 없음). 루브릭 생성 시 `keywords`에 SymPy 파싱 가능 수식 포함 (`RUBRIC_GENERATOR_SYSTEM_MATH`). |
| **국어 (논술)** | sentence-transformers (`jhgan/ko-sroberta-multitask`) + Kiwipiepy | 의미 유사도 30% + **루브릭 기준 커버리지 50%** + 논리 구조 표현 20% |
| **과학** | sentence-transformers + Kiwipiepy | 의미 유사도 40% + 형태소 겹침률 20% + 키워드 적중률 25% + 인과 구조 15% |
| **일반** | 키워드 매칭 | 루브릭 키워드 — 3단계 시맨틱 매칭 (정확↔형태소↔SBERT 구문 레벨) |

> **국어 평가 설계 원칙**: 논술에서 키워드 정확 매칭은 구조적으로 0에 수렴.
> 동일 개념을 다른 표현으로 기술하는 경우를 인정하기 위해 SBERT 기반 의미 커버리지로 설계.

> **시맨틱 키워드 매칭**: `jhgan/ko-sroberta-multitask` 2글자 한국어 단어 임베딩 붕괴 문제를 구문 레벨 비교(`_build_comparison_phrases`)로 해결. 임계값 0.60. 단어↔단어 직접 비교 금지.

### LLM API

| 역할 | 모델 |
|------|------|
| 루브릭 생성 | `claude-opus-4-6` |
| 개별 평가 (심사위원) | `claude-sonnet-4-6` + `gpt-4o-mini`(선택) + `gemini-2.0-flash`(선택) |
| Aggregator / Debate 조율 | `claude-opus-4-6` |

### 데이터베이스
- **Neo4j** (선택적): 학생 답안, LLM별 평가 점수, 교사 수정 피드백을 노드·엣지로 저장.
  드라이버 미설치 시 자동 비활성화.

## 3. 시스템 아키텍처 파이프라인 (LangGraph 기반)

1. **Rubric Generator Node**: 교사의 입력(문항, 모범답안, 총점)을 받아
   Bloom's Taxonomy 기반 엄격한 JSON 루브릭 자동 생성.

2. **Rule-based Router Node**: `subject_tag`에 따라 결정론적 엔진 실행.
   수학(SymPy) / 국어(SBERT + 루브릭 커버리지 + 논리 구조) / 과학(SBERT + 인과 구조).
   실패 시 앙상블으로 폴백 (비블로킹).

3. **Multi-LLM Ensemble Node**:
   - 3개의 LLM 심사위원이 Rule-base 결과를 참조하여 독립 채점 (ThreadPoolExecutor 병렬)
   - Aggregator가 점수 편차를 계산하고, stdev > 1.5 시 Debate 체인 가동 (Claude Opus 조율)
   - 기준(criterion_id)별 편차 재계산 → 편차 큰 기준만 선별 조율

4. **Human-in-the-Loop (HITL)**: FastAPI 세션 방식으로 구현.
   `hitl_ready` SSE 이벤트 → 교사 브라우저에서 검수 패널 표시 →
   교사 승인/수정 → `POST /api/hitl-decision` → 최종 결과 반환.

## 5. 주요 화면 구성 (Screen Specifications)

### 5-1. 평가 설정 및 루브릭 생성 화면 (`/setup`)

- **입력 폼** (`EvaluationForm`): 과목(select), 총점(number), 평가 문항(textarea), 모범 답안(textarea)
- **루브릭 대시보드** (`RubricTable`): 자동 생성된 JSON 루브릭을 shadcn Table로 렌더링.
  셀 클릭 → 인라인 편집(input/textarea 전환). Enter/blur 시 값 확정.
- **채점 시작 버튼**: 수정된 루브릭 + 과목/문항/모범답안(FormMeta)을 sessionStorage에 저장 후 `/grading` 이동.

### 5-2. 학생 답안 채점 및 HITL 검수 대시보드 (`/grading`)

- **학생 답안 입력** (`StudentForm`): 학생 답안 textarea + "채점 시작" 버튼
- **채점 진행 표시** (`GradingProgress`): 4단계 진행 바. SSE progress 이벤트 수신 시 실시간 업데이트.
- **HITL 검수 패널** (`HitlPanel`): `hitl_ready` 이벤트 수신 후 표시. 3개 섹션:

  **섹션 1 — Rule-base 분석 지표**
  - 규칙 기반 점수 바 (과목 배지 포함)
  - 수학: SymPy 동치성 카드 (동치여부 / 검증방법 / 수치통과율 / 대수차이)
  - 국어: NLP 카드 (모범답안 의미 유사도 / 루브릭 기준 커버리지 / 논리 구조 표현)
    - 펼치기: 기준별 커버리지 미니 바 + 논리 구조 카테고리별 ✓△✗ (근거/주장/결론/전개/반론)
  - 과학: NLP 카드 (의미 유사도 / 전문 용어 겹침률 / 키워드 적중률 / 인과 구조)

  **섹션 2 — AI 앙상블 채점 결과** (`EnsembleDetailSection`)
  - 헤더: `ensemble_score / total_score` + Rule-base 기여(10%) 카드 + LLM 기여(90%) 카드
  - 기준 × 모델 점수 테이블:
    - 컬럼: 기준 | 평가기준 | 배점 | Rule(10%) | 모델별 원본 LLM 점수 | [LLM 평균 — 2개+ 모델] | 합산
    - Rule(10%) = `0.1 × rule_ratio × max_score` (점수 단위, 비율 아님)
    - 합산 = `criterion_scores[i].score` 평균 (90%LLM + 10%Rule 이미 적용, ≤ max_score)
    - 합계 행: Rule 기여 합계 + 모델별 원본 합계 + ensemble_score
  - 모델별 상세 피드백 (펼치기)
  - 점수 조율 토론 로그 (펼치기)

  > **점수 구조 원칙**: `ensemble_score`가 Rule 10% + LLM 90%를 이미 내장한 최종 점수.
  > `rule_base_total`(비율 합산)을 가산하면 이중 계산이 되므로 절대 금지.

  **섹션 3 — 교사 검수**
  - 점수 구조 요약: Rule-base 기여 + LLM 기여 + 예상 최종 합계 (`score / total_score`)
  - 최종 점수 조정 input (0 ~ total_score, 기본값 = ensemble_score)
  - 종합 피드백 textarea (편집 가능)
  - 수정 사유 input (선택)
  - [승인] / [수정 후 제출] 버튼 → `POST /api/hitl-decision`
